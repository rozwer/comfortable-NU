# miniSakai / 提出済みタブ

## 機能および要件

- 提出済みエントリをコース別に一覧表示できること
- コースリンクから対象講義へ移動できること
- 締切日時と残り時間を表示できること
- 課題では再提出可否・回数を表示できること
- `+` ボタンから提出済み課題を元にメモを追加できること
- メモ追加時に教科、内容、締切の初期値を引き継げること

## 取得元・Fetch要件

- このタブは追加のネットワーク `fetch` を行わず、miniSakai が保持しているエントリ配列を再利用すること
- 表示対象は `entry.submitted === true` で判定すること
- コース情報は `entities` 側の元データから逆引きして付与すること

## 保存要件

- 読み出し元は `chrome.storage.local[hostname]["Assignments"]` と `chrome.storage.local[hostname]["Quizzes"]` が中心であること
- メモ化操作は `saveNewMemoEntry()` により `chrome.storage.local[hostname]["Memos"]` を更新すること
- 提出済みタブ自体の一覧状態は専用キーを持たず、元データの `submitted` フラグから毎回再計算すること

## 未実装要件

- 提出済みから通常一覧へ戻す正式導線
- 一覧の検索、絞り込み、並び替え
- 提出時刻や最終提出回数の表示
- 提出済みエントリの詳細表示

## 不具合（後ほど調査）

- `submitted` 判定が Assignment と Quiz で一貫しているか確認
- メモ化時の初期値が別講義へずれないか確認
- 件数が多いときの描画速度を確認

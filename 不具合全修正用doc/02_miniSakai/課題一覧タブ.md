# miniSakai / 課題一覧タブ

## 機能および要件

- 課題、クイズ、メモをコース別に一覧表示できること
- 締切に応じて危険度別に色分け表示できること
- コースリンクから対象講義へ移動できること
- 締切日時と残り時間を表示できること
- 課題で遅延提出可否、再提出可否・回数を表示できること
- 完了予定またはチェック用タイムスタンプを表示できること
- `非表示` 操作用ボタンから日付ピッカーモーダルを開けること
- メモ追加ボタンから教科選択、内容入力、締切入力をしてメモを追加できること
- メモを削除できること
- 課題取得時刻、クイズ取得時刻を表示できること
- 未公開課題は設定に応じて表示/非表示を切り替えられること
- 提出済み課題と非表示課題は通常一覧から除外されること

## 取得元・Fetch要件

- エントリ一覧は `getEntities(settings, getSakaiCourses(), cacheOnly)` の結果を使うこと
- 課題取得元は `GET {baseURL}/direct/assignment/site/{courseId}.json`
- クイズ取得元は `GET {baseURL}/direct/sam_pub/context/{courseId}.json`
- メモ取得元は `chrome.storage.local[hostname]["Memos"]` のみで、外部 API は呼ばないこと
- 表示分類は各エントリの `dueTime`, `closeTime`, `openTimeString`, `submitted`, `checkTimestamp` を基準に行うこと
- キャッシュ使用判定は `AssignmentFetchTime`, `QuizFetchTime` と設定値 `cacheInterval` に従うこと

## 保存要件

- `chrome.storage.local[hostname]["Assignments"]`: 課題エントリの保存先
- `chrome.storage.local[hostname]["Quizzes"]`: クイズエントリの保存先
- `chrome.storage.local[hostname]["Memos"]`: メモエントリの保存先
- `chrome.storage.local[hostname]["AssignmentFetchTime"]`: 課題取得時刻
- `chrome.storage.local[hostname]["QuizFetchTime"]`: クイズ取得時刻
- 日付ピッカー保存時は `entry.save(hostname)` を通して、対象エントリが属する保存先へ `checkTimestamp` を書き戻すこと
- メモ追加時は `saveNewMemoEntry()` により `chrome.storage.local[hostname]["Memos"]` を更新すること
- メモ削除時は `removeMemoEntry()` により `chrome.storage.local[hostname]["Memos"]` を更新すること

## 未実装要件

- クイズ・メモにも課題と同等の詳細表示
- 各エントリの詳細モーダル
- 課題をワンクリックで提出済みにする正式導線
- API取得エラー時のリカバリ表示

## 不具合（後ほど調査）

- `非表示` ボタンの意味が UI 上で直感的か確認
- `checkTimestamp` の将来日時運用が時差で崩れないか確認
- 未公開課題の判定が Sakai バージョン差分でずれないか確認
- 課題・クイズ・メモの型差で表示崩れが起きないか確認

## 機能および要件

- Sakai 画面上で miniSakai パネルを表示できること
- ロゴ、バージョン、閉じるボタン、タブ群を表示できること
- `課題一覧 / 提出済み / 非表示 / 設定` タブを切り替えられること
- コース、課題、クイズ、メモを読み込んで保持できること
- 設定変更後に miniSakai 配色と表示内容へ即時反映されること

## 取得元・Fetch要件

- コース一覧は `fetchCourse()` により DOM `.fav-sites-entry` から抽出すること
- 課題は `GET {baseURL}/direct/assignment/site/{courseId}.json` をコースごとに実行して取得すること
- クイズは `GET {baseURL}/direct/sam_pub/context/{courseId}.json` をコースごとに実行して取得すること
- 課題・クイズ取得は `Promise.allSettled` ベースで、一部コース失敗時も残りの成功分で描画継続すること
- メモは外部 `fetch` を行わず、保存済みデータのみを読むこと
- `subSakai` から開いた場合は `cacheOnly = true` として、保存済みデータのみで描画すること

## 保存要件

- `chrome.storage.local["Hostname"]`: 最後に使用した Sakai/TACT ホスト名
- `chrome.storage.local[hostname]["Assignments"]`: 課題一覧の永続化先
- `chrome.storage.local[hostname]["Quizzes"]`: クイズ一覧の永続化先
- `chrome.storage.local[hostname]["Memos"]`: miniSakai メモの永続化先
- `chrome.storage.local[hostname]["Settings"]`: miniSakai 設定の永続化先
- `chrome.storage.local[hostname]["AssignmentFetchTime"]`: 課題 API の最終取得 Unix 時刻
- `chrome.storage.local[hostname]["QuizFetchTime"]`: クイズ API の最終取得 Unix 時刻
- `chrome.storage.local[hostname]["Courses"]`: 保存 API は存在するが、現行描画フローは DOM 抽出を優先していること
- ストレージ構造は `chrome.storage.local[hostname] = { Assignments, Quizzes, Memos, Settings, ... }` の形で統一すること

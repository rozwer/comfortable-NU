# 新着ファイル検知 / 概要

## 機能および要件

- Content API の `modifiedTime` / `createdTime` を前回取得分と比較し、新着を検出すること
- 新着がある教科のサイドバーにバッジ（`cs-notification-badge` 相当）を付けること
- バッジクリック or 該当フォルダ閲覧でバッジを消すこと
- background.ts のアラーム機構で定期チェックすること（設定で間隔変更可能）
- 通知方式は OS 通知ではなくバッジのみであること

## 取得元・Fetch要件

- Content API: `GET {baseURL}/direct/content/site/{siteId}.json`（全コース並列）
- 使用フィールド: `modifiedTime`, `createdTime`, `title`, `type`

## 保存要件

- `chrome.storage.local[hostname]["fileCheckedTimestamps"]`: コース別の最終チェック時刻
- `chrome.storage.local[hostname]["newFileFlags"]`: コース別の新着フラグ（`{ [courseId]: boolean }`）

## UI 配置

- バッジはサイドバーのコースタブに既存の `cs-notification-badge` と同様に表示
- 横断フォルダブラウザ内では新着ファイルに「NEW」バッジを表示

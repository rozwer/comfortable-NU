# 時間割モーダル / 概要

## 機能および要件

- 時間割モーダルを表示できること
- 年度を切り替えられること
- 学期を切り替えられること
- `お気に入りのみ / 全コース` を切り替えられること
- 時間割セルから講義ページへ移動できること
- 教室編集モーダルを開けること
  - 教室名入力
  - 保存
- 教科色設定モーダルを開けること
  - 教科ごとの色変更
  - 個別リセット
  - 全体リセット
  - 出力サイズ選択
  - PNG ダウンロード
- 設定色を時間割へ反映できること

## 取得元・Fetch要件

- 時間割データは基本的に DOM 解析で取得すること
- 第1候補は `a.link-container[title]` から講義タイトル、年度、学期、曜日時限、URL を抽出すること
- 第2候補は `.link-container` から同様の情報を抽出すること
- `全コース` 表示時は `.fav-sites-entry` から全講義候補を抽出すること
- ネットワーク `fetch` には依存せず、ポータル DOM から講義情報を再構成すること
- 表示色は `chrome.storage.local[hostname]["Settings"]` から読むこと

## 保存要件

- `chrome.storage.local[hostname]["TimetableYear"]`: 年度選択の保存先
- `chrome.storage.local[hostname]["TimetableTerm"]`: 学期選択の保存先
- `chrome.storage.local[hostname]["TimetableShowAllCourses"]`: `お気に入りのみ / 全コース` 切替状態の保存先
- `chrome.storage.local[hostname]["cs-timetable-classrooms"]`: 教室編集内容の保存先
- `chrome.storage.local[hostname]["Settings"]`: 時間割配色へ反映する元設定
- `chrome.storage.local[hostname]["cs-timetable-course-colors"]`: 読み出し実装はあるが、現状は保存導線が未接続であること
- `cachedCourses`, `cachedAllCourses` はメモリキャッシュであり、永続化しないこと

## 未実装要件

- 実データ取得失敗時の明確なエラー表示
- 時間割の手動再読み込み
- 曜日・時限のカスタマイズ
- 講師名や教室名の常時表示切替
- PNG 以外の出力形式

## 不具合（後ほど調査）

- ポータル DOM 解析失敗時にサンプルデータへ落ちる挙動が妥当か確認
- 年度/学期保存値が不正な場合の復旧挙動を確認
- 教科色設定プレビューが実際の時間割表示と一致するか確認
- `全コース` 取得で性能問題を起こさないか確認

# 横断事項 / 概要

## 機能および要件

- 設定、キャッシュ、同期、TACT 拡張が相互に矛盾せず動作すること
- Chrome 系ブラウザで安定動作すること

## 取得元・Fetch要件

- Sakai 系 API は主に `/direct/assignment/site/{courseId}.json`, `/direct/sam_pub/context/{courseId}.json`, `/direct/announcement/site/{courseId}.json`, `/direct/content/site/{siteId}.json` を使うこと
- Google 連携は `oauth2/v3/tokeninfo`, `oauth2/v3/userinfo`, `calendar/v3/calendars/primary/events` を使うこと
- 画面情報が API で取得できない箇所は DOM `.fav-sites-entry`, `.link-container`, `#toolMenu ul` などから抽出すること

## 保存要件

- `chrome.storage.local["Hostname"]`: 最終利用ホストの単独キー
- `chrome.storage.local[hostname][...]`: miniSakai、時間割、設定などのホスト別保存領域
- `chrome.storage.local[...]` のフラットキー: カレンダー同期、認証、アラーム関連の共通保存領域
- `localStorage[...]`: TACTメモ、TACTフォルダ、課題/お知らせキャッシュなどのページローカル保存領域
- 現状は `chrome.storage.local` と `localStorage` が混在しているため、将来的な統一方針が必要であること

## 未実装要件

- Safari 対応方針の明確化
- ブラウザ API 抽象化
- ロガー統一
- 型安全性の改善
- キャッシュ戦略の一本化

## 不具合（後ほど調査）

- 大量の `console` 出力の整理
- `calendarSync.ts` の型安全性不足
- `folder-ui.ts` の肥大化による保守性低下

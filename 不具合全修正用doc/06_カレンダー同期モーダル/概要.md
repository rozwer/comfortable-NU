# カレンダー同期モーダル / 概要

## 機能および要件

- Google アカウントでログインできること
- 認証済みアカウント情報を表示できること
- ログアウトできること
- 課題同期、クイズ同期を個別に ON/OFF できること
- 手動同期を開始できること
- 自動同期を ON/OFF できること
- 同期間隔を変更できること
- 最終同期時刻を表示できること
- 最終同期時刻を再取得できること
- 送信済み履歴をクリアできること
- 同期結果や失敗時メッセージを表示できること
- 自動同期時に通知を表示できること

## 取得元・Fetch要件

- 手動同期時の Sakai データ取得元は `GET {baseURL}/direct/assignment/site/{courseId}.json` と `GET {baseURL}/direct/sam_pub/context/{courseId}.json`
- 自動同期時は background から TACT タブへ `getSakaiDataForSync` メッセージを送り、content script 側で課題・クイズを収集すること
- Google 認証状態確認では `https://www.googleapis.com/oauth2/v3/tokeninfo?access_token={token}` を参照すること
- Google アカウント情報取得では `https://www.googleapis.com/oauth2/v3/userinfo` を参照すること
- イベント作成では `https://www.googleapis.com/calendar/v3/calendars/primary/events` を呼ぶこと
- 自動同期可否は background 側の `chrome.alarms` と `lastSyncTime` に従って判定すること

## 保存要件

- `chrome.storage.local["calendarSyncInterval"]`: 自動同期の分単位間隔
- `chrome.storage.local["autoSyncEnabled"]`: 自動同期 ON/OFF
- `chrome.storage.local["lastSyncTime"]`: 最終同期成功時刻
- `chrome.storage.local["sentEventKeys"]`: 送信済みイベントの重複防止キー
- `chrome.storage.local["userAuthenticatedExplicitly"]`: 明示認証済みフラグ
- `chrome.storage.local["lastSyncAttempt"]`: TACT タブなし時の通知抑制用時刻
- ログアウト時は `userAuthenticatedExplicitly`, `oauth_state`, `lastSyncTime`, `sentEventKeys`, `google_auth_token`, `google_user_info`, `calendar_permissions` を削除対象とすること

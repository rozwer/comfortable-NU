# 06_カレンダー同期モーダル 修正事項

## 不具合（コード調査で確認済み）

### BUG-06-01: プロフィール画像URLのXSSリスク [HIGH]
- **場所**: `src/calendarSync.ts` L250
- **内容**: `escapeHtml()` でHTMLタグはエスケープされるが、`<img src="${safePicture}">` でURLスキーム（`javascript:`等）は検証されない
- **対応**: URL先頭が `https://` であることを検証

### BUG-06-02: oauth_stateの残骸コード [LOW]
- **場所**: `src/background.ts` L747
- **内容**: `clearAllAuthenticationData()` で `oauth_state` を削除対象に含むが、認証フローではもう生成していない（意図的な削除後の清掃漏れ）
- **対応**: 不要キーの削除処理を整理

### BUG-06-03: 同期間隔変更後のアラーム遅延 [LOW]
- **場所**: `src/calendarSync.ts` L511-522
- **内容**: 同期間隔を240分→60分に変更した場合、新しい間隔の60分後に初回発火。即座に同期してほしいUX期待とのずれ
- **対応**: 間隔短縮時は即座に1回同期を実行するか、短い初回遅延を設定

### BUG-06-04: 重複イベント作成リスク [MEDIUM]
- **場所**: `src/background.ts` L456-485
- **内容**: `makeEventKey()` が `${type}:${item.id}:${item.title}:${course}` で生成。Sakaiで課題タイトルが変更されると新キーになり、同一課題が重複登録される
- **対応**: キーからtitleを除外し `${type}:${item.id}:${course}` に変更

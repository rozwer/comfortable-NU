# 09_横断事項 修正事項

## 不具合（コード調査で確認済み）

### BUG-09-01: localStorage と chrome.storage.local の混在 [HIGH]
- **場所**: memo.ts(7箇所), file-storage.ts(7箇所), folder-ui.ts(16箇所)
- **内容**: TACTメモ・フォルダ系は `localStorage`（DOM Storage）、miniSakai・カレンダー同期は `chrome.storage.local`（Extension API）を使用。localStorageは拡張機能のリスタートで永続化されないリスクあり。データ管理が二重化
- **対応**: 統一方針の策定。少なくとも重要データ（メモ）はchrome.storage.localへ移行

### BUG-09-02: ストレージ書き込みキューの適用範囲不足 [HIGH]
- **場所**: `src/features/storage/index.ts` L17-68
- **内容**: `toStorage()` の書き込みキューは `writeQueues` Map で直列化されるが、`calendarSync.ts`, `background.ts`, `content_script.ts` は直接 `chrome.storage.local.set()` を呼ぶ。キューをバイパスし、レースコンディションのリスク残存
- **対応**: 全 `chrome.storage.local` 操作を `toStorage()` 経由に統一するか、グローバルキューを導入

### BUG-09-03: 統一ロガーなし・過剰なconsole出力 [HIGH]
- **場所**: 全ソースファイル（213箇所）
- **内容**: `background.ts`(78箇所), `timetable.ts`(50箇所), `folder-ui.ts`(40箇所) 等。`[DEBUG]`, 絵文字マーカー等が混在。本番ビルドでも全出力される
- **対応**: ログレベル付きロガーの導入。本番ビルドではDEBUGレベルを除外

### BUG-09-04: ルートの background.js が管理不明 [MEDIUM]
- **場所**: `/background.js`（44KB）
- **内容**: webpack出力のコンパイル済みファイルがリポジトリルートに存在。`src/background.ts` が正しいソースだが、ルートファイルとの関係が不明瞭
- **対応**: `.gitignore` に追加するか、ビルド出力ディレクトリを明確化

### BUG-09-05: calendarSync.ts の型安全性不足 [MEDIUM]
- **場所**: `src/calendarSync.ts` 複数箇所
- **内容**: `response.accounts || []` の型アサーションなし、`assignment.flatMap()` のnullチェックなし、日時比較で undefined の可能性あり
- **対応**: strict null チェックの強化、入力バリデーション追加

### BUG-09-06: folder-ui.ts の肥大化（1720行） [MEDIUM]
- **場所**: `src/features/tact/folder-ui.ts`
- **内容**: 単一クラスにタブ切替、ツリー描画、ダウンロード、課題、お知らせ、キャッシュ、移動操作が集約。保守性・テスト容易性が低い
- **対応**: FolderTreeRenderer, FolderCacheManager, FolderDownloadManager 等への分割
